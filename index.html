How it works
chitnow — agent-hook — 80×24
>
Agent runs a risky command
rm -rf, git push --force, sudo, etc. — the hook intercepts before execution.
>
Relay wakes your iPhone
The optional relay sends only a generic wake-up push. Your iPhone fetches the full request from the LAN broker and pings the Watch.
>
You approve or deny
Review the full command in ChitNow, then approve or deny. The agent proceeds or stops immediately.
>
Command data stays on your LAN
Relay and APNs receive only a generic approval event. Commands, broker URLs, API keys, and decisions stay between your devices and Mac broker.
Setup
1

Install the Mac Helper

One script sets up everything: the broker, the Claude Code and Codex hooks, the recommended Codex high-risk rules, auto-start, and codexbar (via Homebrew) for token usage, cost, and quota display. It finishes by opening the secure pairing page.

$git clone https://github.com/WiwaxiaRay/thenow && cd thenow
$bash install.sh

Requires Python 3.11+. No Homebrew? Install codexbar manually and keep it running. For generic push wake-ups, deploy the optional Cloudflare relay first, then run CHITNOW_RELAY_URL=https://your-worker.workers.dev bash install.sh.

2

Pair the iPhone app

The pairing page is already open in your browser. Open ChitNow on your iPhone and scan the one-time QR code.

Note: Your browser will show a certificate warning — this is expected because the cert is generated locally on your Mac. Keep the setup-token URL private, and use the URL printed by your own installer.

3

Trust the Codex hook Codex only

The installer has already configured Codex. Open the Codex TUI, run /hooks, and trust the ChitNow hook. Claude Code-only users can skip this step.

This one-time trust confirmation is a Codex security boundary and is intentionally not automated. If the Watch does not respond within 15 seconds, Codex falls back to its native approval UI.

// notes
📡 LAN only. Mac, iPhone, and Watch must be on the same Wi-Fi network. Switching networks or a VPN may require re-pairing.
🔔 The optional Cloudflare relay sends a generic wake-up push to the iPhone. Without the relay, foreground polling remains available, but the Watch app must be open for reliable approval delivery.
🔒 The relay and Apple APNs receive only a generic approval-pending event. Commands, summaries, broker URLs, API keys, fingerprints, and approval decisions never pass through the relay.
🔑 Relay requests use per-installation HMAC credentials, timestamps, and nonce replay protection. Installation credentials can be rotated or revoked.

Ready to pair?

Run the installer. It opens the private pairing page automatically.

bash install.sh
Browser will warn about the certificate — this is normal for a local self-signed cert.
Click Advanced → Proceed to continue.
