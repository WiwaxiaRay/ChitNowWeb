When Claude Code or Codex wants to run a high-risk command, ChitNow pauses and waits for you to approve or deny — right on your wrist.

macOS watchOS Claude Code Codex Local-first
How it works
chitnow — agent-hook — 80×24
>
Agent runs a risky command
rm -rf, git push --force, sudo, etc. — the hook intercepts before execution.
>
Apple Watch vibrates
A notification appears with the full command. Tap to see details inline.
>
You approve or deny
One tap on the Watch. The agent proceeds or stops immediately. No unlock required.
>
Fully local — nothing leaves your Mac
The broker runs on your machine. APNs delivers the push; no cloud server stores your commands.
Setup
1

Install the Mac Helper

Clone the repo and start the broker. It runs as a background service and auto-starts on login.

$git clone https://github.com/WiwaxiaRay/thenow
$cd thenow/broker && python3 -m venv .venv && .venv/bin/pip install -r requirements.txt
$launchctl load ~/Library/LaunchAgents/com.wangyang.thenow-broker.plist
2

Install the iPhone app

Download ChitNow from TestFlight or install via Xcode. The Watch app installs automatically alongside it.

3

Pair your iPhone

Open the pairing page on your Mac browser, then scan the QR code in the ChitNow iPhone app.

$open https://localhost:8000/pair

Note: Your browser will show a security warning for the self-signed certificate. Click Advanced → Proceed to localhost (Chrome) or Show Details → visit this website (Safari). This is expected.

4

Wire up the hook

Claude Code — the hook is configured automatically via ~/.claude/settings.json.
Codex — add the hook to ~/.codex/config.toml and re-trust it in the Codex TUI.

See the README for exact config snippets.

Ready to pair?

Make sure the Mac Helper is running, then open the pairing page.

open https://localhost:8000/pair
Browser will warn about the certificate — this is normal for a local self-signed cert.
Click Advanced → Proceed to continue.
